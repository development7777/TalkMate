## Reporting a Vulnerability

We take the security of TalkMate seriously. If you believe you have found a security vulnerability, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to [kodai819@outlook.com](mailto:kodai819@outlook.com).

You should receive a response within 48 hours. If for some reason you do not, please follow up via email to ensure we received your original message.

Please include the following information in your report:

- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

This information will help us triage your report more quickly.

## What to expect

After you submit your report, we will:

1. Confirm the problem and determine the affected versions
2. Audit the code to find any similar problems
3. Prepare fixes for all supported versions

## Security Updates

Security updates will be released as patch versions (e.g., 1.0.1, 1.0.2, etc.) for the latest supported version.

## Best Practices

To help ensure the security of your TalkMate installation:

1. Always use the latest version of TalkMate
2. Keep your dependencies up to date
3. Follow security best practices for your deployment environment
4. Regularly check for security updates

## Disclosure Policy

When we receive a security bug report, we will:

1. Confirm the problem and determine the affected versions
2. Audit the code to find any similar problems
3. Prepare fixes for all supported versions
4. Release new versions with the fixes
5. Credit the reporter in the release notes
