# TalkMateへの貢献

TalkMateへの貢献に興味を持っていただき、ありがとうございます！このドキュメントでは、プロジェクトへの貢献方法についてのガイドラインと手順を説明します。

## 行動規範

貢献する前に、[行動規範](CODE_OF_CONDUCT_ja.md)を必ずお読みください。すべての貢献者がこれらのガイドラインに従うことを期待しています。

## 貢献方法

### バグの報告

1. Issuesセクションで既に報告されていないか確認してください
2. 報告されていない場合は、明確で説明的なタイトルで新しいissueを作成してください
3. バグ報告テンプレートを使用し、以下の情報をできるだけ詳しく提供してください：
   - バグの再現手順
   - 期待される動作
   - 実際の動作
   - 該当する場合はスクリーンショット
   - 環境情報（OS、ブラウザなど）

### 機能の提案

1. Issuesセクションで既に提案されていないか確認してください
2. 提案されていない場合は、明確で説明的なタイトルで新しいissueを作成してください
3. 機能リクエストテンプレートを使用し、以下を提供してください：
   - 機能の明確な説明
   - この機能が有用である理由
   - 追加のコンテキストや例

### プルリクエスト

1. リポジトリをフォークする
2. 機能やバグ修正用の新しいブランチを作成する
3. 変更を加える
4. 必要に応じてテストを書くまたは更新する
5. すべてのテストが通ることを確認する
6. 必要に応じてドキュメントを更新する
7. プルリクエストを提出する

### 開発環境のセットアップ

リポジトリをクローン：

   ```bash
   git clone https://github.com/development7777/TalkMate.git
   cd TalkMate
   ```

### コーディング規約

- 既存のコードスタイルに従う
- 意味のある変数名と関数名を使用する
- 明確で簡潔なコメントを書く
- 関数は小さく、焦点を絞ったものにする
- 新機能やバグ修正にはテストを書く

### コミットメッセージ

- 明確で説明的なコミットメッセージを使用する
- 現在形の動詞で始める（例：「追加」、「修正」、「更新」）
- 最初の行は50文字以内に収める
- 必要に応じて詳細な説明を追加する

### ドキュメント

- 新機能の追加や重要な変更を行った場合はREADME.mdを更新する
- 複雑なコードセクションにはコメントを追加する
- エンドポイントを変更した場合はAPIドキュメントを更新する

## ヘルプの入手

ヘルプが必要な場合や質問がある場合：

1. [ドキュメント](README.md)を確認する
2. 既存のissueを検索する
3. 質問が回答されていない場合は新しいissueを作成する

## ライセンス

TalkMateに貢献することで、あなたの貢献はプロジェクトの[LICENSE](LICENSE)ファイルの下でライセンスされることに同意したことになります。

TalkMateへの貢献ありがとうございます！
